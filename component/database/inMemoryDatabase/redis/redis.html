Redis

Redis

简介

Redis是Remote Dictionary Server的缩写，其支持的类型有5种，包括String: 字符串、Hash: 散列、List: 列表、Set: 集合、Sorted Set: 有序集合。

安装使用

1.安装 redis是绿色安装的方式，可以直接下载redis安装包并解压执行

./redis-server #使用默认配置文件，也可添加.conf配置文件

2.redis.conf文件

daemonize no #是否已守护进程方式运行，默认为no
port 6279 #制定redis监听端口
logevel notice #redis支持4种日志级别，debug、verbose、notice、warning
databases 16 #设置数据库的数量，数据库之间数据不能共享，并且只有单机才有
dbfilename dump.rdb #指定本地数据库文件名
dir ./ #指定本地数据库存放目录
replicaof <masterip> <masterport> #当本机为slave时，设置master的IP地址及端口，也可以为slaveof
masterauth <master-password> #当master设置了密码保护时，slave需设置密码
requirepass <password> #设置Redis连接密码，如果配置了连接密码，客户端在连接Redis时需要通过AUTH<password>命令提供密码
maxclients 128 #设置同一时间最大客户端连接数
maxmemory <bytes> #指定Redis最大内存限制
maxmemory-policy #当数据达到最大内存限制时执行的淘汰策略，redis支持六种策略，分别为volatile-lru、volatile-ttl、volatile-random、allkeys-lru、allkeys-random和no-enviction(默认)。
appendonly no #是否每次更新后操作后进行日志记录，默认为异步写
appendfilename appendonly.aof #指定更新日志文件名
appendfsync everysec #指定更新条件，有no、always和everysec(默认)选项
include #指定包含其它的配置文件

3.补充 redis连接，可以用使用redis-cli命令

redis-cli -h host -p port -a password

对于备份数据的恢复，可以直接将备份文件(默认为dump.rdb)移动到redis安装目录并启动服务即可。 对于redis的性能测试，可以使用redis的redis-benchmark工具

redis-benchmark -c num -n num -q #指定在多少并发下发送多少请求

高可用方案

目前redis的高可用方案主要有三种，分别为基于keepalive的主从复制、sentinel哨兵以及redis cluster，其中第一种方式在keepalive进行描述，这里不做过多说明，只对JAVA的调用简答描述

public static void main(String[] args) {
    Jedis jedis_M = new Jedis("127.0.0.1",6379);
    Jedis jedis_S = new Jedis("127.0.0.1",6380);
    //从机连接到主机
    jedis_S.slaveof("127.0.0.1",6379);
    //主机写入
    jedis_M.set("class","1122");
    //从机读取
    String result = jedis_S.get("class");
}

1.sentinel哨兵 Sentinel架构中包含若干个Sentinel节点和Redis数据节点，每个Sentinel节点会对数据节点和其余Sentinel节点进行监控，当它发现节点不可达时，会对节点做下线标识。如果被标识的为主节点，sentinel节点将进行“协商”，当大多数Sentinel节点都认为主节点不可达时，它们会选举出一个Sentinel节点来完成自动故障转移的工作，并将这个变化实时通知给Redis应用方。 sentinel.conf

port 26379  #sentinel实例运行的端口 默认26379
protected-mode no #允许所有地址的访问，如果是yes则只允许本机，默认为yes
sentinel monitor <master-name> <ip> <redis-port> <quorum>  #sentinel监控的redis主节点的ip port，quorum是当这些数量的sentinel认为master主节点失联，此时认为主节点下线
sentinel auth-pass <master-name> <password> #设置sentinel连接主从的密码
sentinel parallel-syncs <master-name> <num> #同一时刻最多有多少个slave对新master进行复制
sentinel down-after-milliseconds <master-name> <milliseconds> #指定多少毫秒主节点没有应答sentinel，则认为主节点下线，默认30秒
sentinel known-replica <master-name> <ip> <redis-port> #其他sentinel节点的信息

sentinel的启动

redis-sentinel sentinel.conf

sentinel状态查看

redis-cli -h ip -p port info Sentinel #通过这里可以查询从的数量和sentinel的数量，如果sentinel数量不对，考虑是否是myid配置的问题

JAVA连接redis哨兵模式的代码

JedisPoolConfig jedisPoolConfig = new JedisPoolConfig();
jedisPoolConfig.setMaxTotal(10);
jedisPoolConfig.setMaxIdle(5);
jedisPoolConfig.setMinIdle(5);    
// 哨兵信息
Set<String> sentinels = new HashSet<String>(Arrays.asList("192.168.11.128:6379", "192.168.11.129:6379", "192.168.11.130:6379"));
//创建连接池，mymaster是我们配置给哨兵的服务名称,123是连接redis服务器的密码
JedisSentinelPool pool = new JedisSentinelPool("mymaster", sentinels, jedisPoolConfig, "123");
//获取客户端
Jedis jedis = pool.getResource();
//执行两个命令
jedis.set("mykey", "myvalue");

2.redis cluster

  • redis集群采用P2P模式，是完全去中心化的，不存在中心节点或者代理节点；
  • 如果集群中任意一个节点挂了，并且该节点没有从节点（备份节点），那么这个集群就挂了。

redis cluster安装简述

1.首先修改redis.conf中cluster-enabled yes
2.安装Ruby，并执行./redis-trib.rb create --replicas num -a password ip:port
#需要说明的是目前无法指定哪些节点为主节点，哪些为从节点
cluster info #集群信息查看

JAVA连接redis集群模式代码

public static void main(String[] args) {
    JedisPoolConfig poolConfig = new JedisPoolConfig();  
    poolConfig.setMaxTotal(1);  // 最大连接数   
    poolConfig.setMaxIdle(1);  // 最大空闲数 
    poolConfig.setMaxWaitMillis(1000); // 最大允许等待时间
    Set<HostAndPort> nodes = new LinkedHashSet<HostAndPort>();
    nodes.add(new HostAndPort("IP", PORT));
    nodes.add(new HostAndPort("IP", PORT)); //至少需要6个
    JedisCluster cluster = new JedisCluster(nodes, poolConfig);
    String name = cluster.get("name");
    cluster.set("age", "18");
    try {
        cluster.close();
    } catch (IOException e) {
        e.printStackTrace();
    }
}
